### 小团体用户

用户每天：

# 八、核心业务逻辑

## 8.1 登录流程

1. 用户打开小程序
2. 调用 `wx.login()`
3. 获取 code
4. 发送到服务器
5. 服务器换取 openid
6. 创建用户
7. 返回 token

---

## 8.2 步数同步流程

1. 点击刷新
2. 调用 `wx.getWeRunData()`
3. 获取 encryptedData
4. 发送到服务器
5. 服务器解密
6. 得到 30 天步数
7. 更新用户每日步数
8. 更新成长等级

---

## 8.3 步数处理逻辑

微信返回：

最近 30 天步数数组

存储方式：

表 `user_step_daily`

逻辑：

- 如果当天数据存在，则更新最大值
- 如果不存在，则创建记录

用户总步数计算：

`sum(user_step_daily.step_count)`

本周步数计算：

统计本周日期范围内的步数总和。
