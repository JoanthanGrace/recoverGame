# 微信小游戏《打工人康复指南》MVP 版本 PRD

**文档版本:** v1.0
**更新日期:** 2026-03-05
**开发引擎:** Cocos Creator (2D)
**目标平台:** 微信小游戏

---

## 一、 产品概述

- **产品定位:** 极简医学解谜与科普类微信小游戏。
- **核心体验:** 观察病理症状 ➔ 识别肌肉状态 ➔ 拖拽对应康复工具 ➔ 治愈通关获得正向反馈。
- **MVP 目标:** 跑通“核心交互循环”与“结算分享界面”，验证玩法可行性与分享裂变能力。
- **核心知识点:** 将真实的解剖学逻辑（紧张肌肉需放松，无力肌肉需强化）包装成游戏解谜元素。

---

## 二、 核心界面布局 (UI Layout)

游戏采用全屏竖版显示（适配主流手机屏幕），自上而下划分为三个核心层级：

### 1. 顶部信息栏 (Top Bar - 占屏 15%)

- **关卡标题:** 居中或左上角显示，例如 `Lv.1 键盘侠的觉醒`。
- **体态健康条:** 位于标题下方。初始状态为 0%（红色填充），满分为 100%（绿色填充）。
- **提示按钮:** 右上角放置 `?` 图标，点击弹出解剖学提示文本（防卡关机制）。

### 2. 中部主操作区 (Main Play Area - 占屏 60%)

- **人物立绘:** 屏幕正中央放置病人 2D 漫画图像。
- **肌肉透视热区:** - **热区 A (紧张肌肉):** 呈现**红色闪烁**，代表“紧张缩短”，视觉上需要有“僵硬感”。
  - **热区 B (无力肌肉):** 呈现**蓝色暗淡**，代表“无力拉长”，视觉上需要有“松弛感”。

### 3. 底部工具栏 (Tool Dock - 占屏 25%)

- **布局:** 底部居中，横向排列的方形物品槽位。
- **工具图标:** 放置可拖拽的康复工具（如筋膜球、弹力带等），需附带极简的 UI 提示词。

## 三、 关卡 1 详细设计：键盘侠的觉醒

### 1. 关卡背景设定

- **病人画像:** 因常年伏案写代码/打游戏，导致严重含胸驼背（上交叉综合征）的打工人。
- **病理逻辑:** 前侧胸大肌紧张缩短，后侧菱形肌/中下斜方肌无力拉长。

[Image of Upper Crossed Syndrome anatomy diagram]

### 2. 交互逻辑与判定规则

本关卡需完成两次正确的拖拽操作才能通关。系统需实时进行碰撞检测与状态判定：

| 玩家操作                             | 系统判定                  | 视觉与听觉反馈 (Feedback)                                     |
| :----------------------------------- | :------------------------ | :------------------------------------------------------------ |
| 拖拽【筋膜球】至前胸【胸大肌】发红处 | **判定正确** (放松)       | 播放“气球放气”舒缓音效；红光消失；人物肩膀微张；健康条 +50%。 |
| 拖拽【弹力带】至后背【菱形肌】发蓝处 | **判定正确** (强化)       | 播放“叮”升级音效；蓝光变为金光；人物挺直腰板；健康条满 100%。 |
| 拖拽【筋膜球】至后背【菱形肌】       | **判定错误** (工具不匹配) | 播放“错误/警报”音效；屏幕轻微震动；工具自动弹回底部槽位。     |
| 拖拽【弹力带】至前胸【胸大肌】       | **判定错误** (工具不匹配) | 播放“错误/警报”音效；屏幕轻微震动；工具自动弹回底部槽位。     |
| 将工具拖拽到非发光判定区域           | **无效操作**              | 工具无缝弹回底部槽位，无额外惩罚提示。                        |

## 四、 结算与社交分享机制 (Settlement & Social)

当顶部健康条达到 100% 时，立即锁定玩家操作，弹出结算面板。

### 1. 结算面板元素

- **通关文案:** “治疗成功！胸肌松一松，背肌紧一紧，告别猥琐颈！”
- **知识沉淀 (科普小贴士):** “上交叉综合征：长期伏案导致前侧胸肌紧张，后侧背肌无力。需松前侧，强后侧。”

### 2. 社交裂变 (分享点)

- **分享按钮:** 醒目的【分享给受苦的群友】绿色按钮。
- **接口调用:** 调用微信小游戏原生 `wx.shareAppMessage` 接口。
- **分享卡片设计:** - **文案:** “我是第 X 个治好键盘侠的神医，快来测测你的体态！”
  - **配图:** 游戏通关后的搞笑对比图或极具冲击力的标题图。

### 3. 后续流程

- 增加【下一位病人】按钮。MVP 阶段点击后可提示“更多疑难杂症正在送医途中，敬请期待...”。
